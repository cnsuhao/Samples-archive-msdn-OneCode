How to use Membership Provider in Self-database(VBASPNETMembershipProvider)

Introduction

This demo is mainly showing how to use ASP.NET 2.0 (since) new featureMembership Provider and to generate all of its related tables and combine them into your own SQL databaseThe most difference is that in MSDNyou can see it has implemented some classes or interfaces to do your own waycompared with thiswhat we do is just use aspnet_regsql.exe as well as some specific tools to generated related database's tables into your own databaseand then change the Membership Provider to your own DB's connectionwhich means all of the functions are inherited from Microsoft's onesSo you don't need to write any codes about membership but just call them

Running the Sample

Please follow these demonstration steps below.

Step 1: Open the VBASPNETMembershipProvider.sln. Expand the VBASPNETMembershipProvider web application. Right click CreateUser.aspx and choose "View in Browser" to create a new user.

 

 [Note] You may need to install the SQL Server 2008 r2 express. You can download it here: http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=3743

Step 2: Then press Ctrl + F5 to show the Default.aspx.

Step 3:   Validation finished.

Using the Code

Code Logical:

Step 1. Create a new database in SQL Server. Name it as MembershipProviderTest".   

Step 2. Use the command-line tool aspnet_regsql.exe (located in the Framework folder) to create the tables and procedures used by the Membership Provider.

Step 3. Create a VB "ASP.NET Web Application" in Visual Studio 2008. Name it as VBASPNETMembershipProvider".

Step 4.  Add five Page then rename them to CreateUser.aspx, Login.aspx,ChangePassword.aspx, RecoveryPassword.aspx and Default.aspx.

            Add a CreateUserWizard Control to the CreateUser page, the code as show below:

  - VB code snippet -
 
<asp:CreateUserWizard ID="CreateUserWizard1" runat="server" ActiveStepIndex="0" ContinueDestinationPageUrl="~/Default.aspx">
        <WizardSteps>
          <asp:CreateUserWizardStep ID="CreateUserWizardStep1" runat="server">
          </asp:CreateUserWizardStep>
          <asp:CompleteWizardStep ID="CompleteWizardStep1" runat="server">
          </asp:CompleteWizardStep>
        </WizardSteps>
      </asp:CreateUserWizard>
 
- end -

         Add a Login Control to the Login page, the code as show below:

- VB code snippet -
 
<asp:Login ID="Login1" runat="server">
       </asp:Login>
 
- end -

Add a ChangePassword Control to the ChangePassword page.    
Add a PasswordRecovery Control to the PasswordRecoverypage.

Finally, add a LoginStatus Control to the Default page.

Step 5.  Now, begin to edit the web.xml.

First, configure the Connection Strings as show below; you can try the connectionString1 or connectionString2.

- XML code snippet -
 
<connectionStrings>
    <!-- Clear the default connection. -->
    <clear/>
 
    <!-- Connect to the database in the App_Data folder -->
    <add name="LocalSqlServer" providerName="System.Data.SqlClient" connectionString="Data Source=(LocalDB)\v11.0;AttachDbFilename=|DataDirectory|\MembershipProviderTest.mdf;Integrated Security=True "/>
 
    <!-- Connect to the database which has been attached to the database manager. -->
    <!--<add name="LocalSqlServer" connectionString="Data Source=.;Initial Catalog=D:\PROJECTDIR\NET\CSASPNETMEMBERSHIPPROVIDER\VBASPNETMEMBERSHIPPROVIDER\APP_DATA\MEMBERSHIPPROVIDERTEST.MDF;Integrated Security=True"/>-->
  </connectionStrings>
 
- end -


Second, configure the CreateUser page and the security authentication mode:

- XML code snippet -
 
<location path="CreateUser.aspx">
   <system.web>
     <authorization>
       <allow users="?" />
     </authorization>
   </system.web>
 </location>
 

<!--

            The <authentication> section enables configuration

            of the security authentication mode used by

            ASP.NET to identify an incoming user.

        -->

    <authentication mode="Forms" >

      <forms loginUrl="login.aspx"

        name=".ASPXFORMSAUTH"  />

    </authentication>

 

    <authorization>

      <deny users="?" />

    </authorization>

 
- end -

 Finally, configure the membership to AspNetSqlMembershipProvider:

- XML code snippet -
 
<!--Build-in Provider Configuration-->
 
  <membership defaultProvider="AspNetSqlMembershipProvider" userIsOnlineTimeWindow="15">
    <providers>
    </providers>
  </membership>
  <!--Build-in Provider Configuration-->
 
- end -

[Note] If you want to use Recovery Password, you must configure mailSettings your own:

- XML code snippet -
 
<system.net>
   <mailSettings>    
     <smtp deliveryMethod="Network" from="*@server.com">
       <network host="*" password="*****" port="25" userName="*****" defaultCredentials="true"/>
     </smtp>
   </mailSettings>
 </system.net>
 
- end -


Step 6. When targeting .NET 4.5 we enable Unobtrusive Validation by default. You need to have jQuery in your project and have something like this in Global.asax to register jQuery properly:

- VB code snippet -
 
Dim myScriptResDef As New ScriptResourceDefinition()
       myScriptResDef.Path = "~/JS/jquery-1.8.1.js"
       myScriptResDef.DebugPath = "~/JS/jquery-1.8.1.js"
       myScriptResDef.CdnPath = "http://code.jquery.com/jquery-1.8.1.js"
       myScriptResDef.CdnDebugPath = "http://code.jquery.com/jquery-1.8.1.js"
       ScriptManager.ScriptResourceMapping.AddDefinition("jquery", Nothing, myScriptResDef)
 
- end -

Add global.asax item in your project and add above code to Application_Start. Replacing the version of jQuery with the version you are using.
Step 7. Build the application and you can debug it.
